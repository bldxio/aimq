# Knowledge Systems

A structured approach to capturing, organizing, and evolving project knowledge. Think of it as a "second brain" for your project that grows with every lesson learned.

- **[Knowledge Systems Templates](./knowledge-systems-templates.md)** - Standard templates for patterns, standards, architecture, quick references

### Core Values
1. **Living Documentation** - Evolves with the project
2. **Small and Focused** - Files under 400 lines
3. **Linked Knowledge** - Build a knowledge graph
4. **Shallow Structure** - No nested directories
5. **Continuous Cultivation** - Regular maintenance

### Gardening Workflow
1. **Capture (Daily)** - Use `/remember` to capture insights
2. **Learn (Weekly)** - Use `/learn` to extract lessons
3. **Organize (Monthly)** - Use `/cultivate` to maintain
4. **Explore (As Needed)** - Use `/focus` and `/levelup`
